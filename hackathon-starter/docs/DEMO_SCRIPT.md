# Demo Script

Use this reusable outline for a polished 3-5 minute hackathon demo.

## Introduction

Time: 20-30 seconds

"Hello, we are [Team Name], and today we are presenting [Project Name], a project built to help [target users] solve [core problem]."

## Problem

Time: 30-45 seconds

"Teams and users often struggle with [problem]. Existing approaches are slow, fragmented, or hard to use during real-world time pressure."

## Solution

Time: 30-45 seconds

"We built [Project Name], which helps users [primary outcome]. Our goal was to create a simple experience that makes [key benefit] faster and more reliable."

## Architecture

Time: 30-45 seconds

"The project is built on a reusable Next.js starter with shared UI primitives, common components, typed utilities, centralized configuration, and global providers. That structure helped us move quickly while keeping the code organized."

## Live Demo

Time: 90-150 seconds

Suggested flow:

1. Start on the main landing or entry screen.
2. Show the primary user journey from start to finish.
3. Highlight one or two details that demonstrate polish, speed, or usability.
4. Show any important edge case, admin flow, or differentiator only if it helps the story.

Suggested narration:

"Here is the core workflow. First, the user [step one]. Next, they [step two]. Finally, they [step three], which produces [outcome]."

## Tech Stack

Time: 20-30 seconds

"We built this with Next.js, React, TypeScript, Tailwind CSS, Zod, React Hook Form, and Sonner, using reusable shared components to keep development fast."

## Challenges

Time: 20-30 seconds

"One challenge was [technical or product challenge]. We addressed it by [brief solution], which let us keep the experience stable and demo-ready."

## Future Scope

Time: 20-30 seconds

"With more time, we would expand into [next feature], improve [area], and validate the product with more real users."

## Closing

Time: 10-20 seconds

"Thank you for your time. We would love to show the repository, architecture, and next steps during questions."
