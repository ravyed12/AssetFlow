# Submission Checklist

Use this checklist right before you submit the project.

## Final Submission Review

- [ ] Repository is updated with the latest stable code.
- [ ] README is completed with the final problem statement, solution, architecture, and links.
- [ ] Screenshots are added to the repository or README.
- [ ] Environment variables are configured for the final demo environment.
- [ ] `npm run build` completes successfully.
- [ ] `npm run lint` completes successfully.
- [ ] Evaluator or reviewer access has been added to the GitHub repository if the event requires it.
- [ ] Demo video has been recorded.
- [ ] Demo video has been uploaded and is accessible.
- [ ] Repository link has been submitted in the event portal or form.
- [ ] Live demo link has been verified.
- [ ] Team member details are accurate wherever the event requires them.
- [ ] One teammate has completed a final walkthrough without local-only dependencies.

## Final Verification

- [ ] README links work.
- [ ] Demo assets are visible without special permissions.
- [ ] The repository contains no secrets.
- [ ] The judged workflow can be explained in under five minutes.
- [ ] The team knows who will present and who will answer technical questions.
